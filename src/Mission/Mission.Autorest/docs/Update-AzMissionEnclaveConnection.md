# Update-AzMissionEnclaveConnection

## SYNOPSIS
Update a EnclaveConnectionResource

## SYNTAX

### UpdateExpanded (Default)
```
Update-AzMissionEnclaveConnection -Name <String> -ResourceGroupName <String> [-SubscriptionId <String>]
 [-SourceCidr <String>] [-Tag <Hashtable>] [-DefaultProfile <PSObject>] [-AsJob] [-NoWait] [-Confirm]
 [-WhatIf] [<CommonParameters>]
```

### UpdateViaIdentityExpanded
```
Update-AzMissionEnclaveConnection -InputObject <IMissionIdentity> [-SourceCidr <String>] [-Tag <Hashtable>]
 [-DefaultProfile <PSObject>] [-AsJob] [-NoWait] [-Confirm] [-WhatIf] [<CommonParameters>]
```

### UpdateViaJsonFilePath
```
Update-AzMissionEnclaveConnection -Name <String> -ResourceGroupName <String> -JsonFilePath <String>
 [-SubscriptionId <String>] [-DefaultProfile <PSObject>] [-AsJob] [-NoWait] [-Confirm] [-WhatIf]
 [<CommonParameters>]
```

### UpdateViaJsonString
```
Update-AzMissionEnclaveConnection -Name <String> -ResourceGroupName <String> -JsonString <String>
 [-SubscriptionId <String>] [-DefaultProfile <PSObject>] [-AsJob] [-NoWait] [-Confirm] [-WhatIf]
 [<CommonParameters>]
```

## DESCRIPTION
Update a EnclaveConnectionResource

## EXAMPLES

### Example 1: Patch an enclave connection's tags
```powershell
Update-AzMissionEnclaveConnection -Name 'contoso-connection' -ResourceGroupName 'mission-rg' -Tag @{ environment = 'production' }
```

```output
Name               Location ResourceGroupName ProvisioningState
----               -------- ----------------- -----------------
contoso-connection eastus   mission-rg        Succeeded
```

Updates only the tags on the existing `contoso-connection` enclave connection, leaving its endpoints unchanged (PATCH semantics).
