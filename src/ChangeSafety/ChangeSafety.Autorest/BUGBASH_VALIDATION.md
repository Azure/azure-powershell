# ChangeSafety bug-bash validation

This document captures the first ChangeSafety bug-bash batch fixed by PR #7.

## Fixes included

| Area | Fix |
|---|---|
| ChangeRecord target validation | Added shared validation for `-Targets` input shape, required target identifiers, valid `httpMethod`, valid `ChangeType`, valid `RolloutType`, and anticipated time windows. |
| Required parameters | Marked `ChangeType`, `RolloutType`, and `Targets` mandatory for the `New-AzChangeSafetyChangeRecord -Targets` parameter set. |
| Past time validation | Rejects `AnticipatedStartTime` / `AnticipatedEndTime` values in the past before calling the service. |
| HTTP method validation | Rejects invalid `Targets.httpMethod` values and documents allowed verbs: `DELETE`, `GET`, `HEAD`, `PATCH`, `POST`, `PUT`. |
| Target rendering | Makes `ChangeDefinitionDetail` and target details visible in `Get-AzChangeSafetyChangeRecord` output. |
| Output formatting | Updates ChangeRecord formatting so `AnticipatedStartTime` appears before `AnticipatedEndTime`, and the default table includes useful fields. |
| Pipeline chaining | Fixes the validated bug-bash chaining case: `New-AzChangeSafetyChangeRecord ... | Get-AzChangeSafetyChangeRecord -ResourceGroupName ...`. This is scoped to ChangeRecord output to `Get-AzChangeSafetyChangeRecord`; broader `Update` or `Remove` chaining is not claimed in this PR. |
| RolloutType immutability | Blocks `Update-AzChangeSafetyChangeRecord -Targets -RolloutType ...` with a clear local error after creation. |
| Single target serialization | Preserves a single target as a JSON array when updating `-Targets`. |
| StageMap lookup | Adds a fallback for `Get-AzChangeSafetyStageMap -Name` so name-only lookup can find resource-group scoped StageMaps created by `New-AzChangeSafetyStageMap`. |
| Tests | Adds and updates playback tests covering the fixed behaviors. |
